# 观察池（独立维护）

> 本表独立于「基金模拟投资方案.md」与「持仓速览表.md」，是观察池的**唯一权威来源**。
> 更新口径：随每日/每次「查潜力」「更新今天」的诊断刷新「当前信号 / 状态 / 备注」。

## 表格说明
- **观察原因**：当初为什么把它放进观察池（逻辑锚点，移除时回看）。
- **状态**：`核心观察`（每日盯）/ `候补`（暂移出每日观察，待条件修复再纳回）/ `已移除`。
- **「核心观察」的四种角色**（都每日盯，但当前用途不同）：
  - `核心观察`（主力候选）：随时可能低位建仓/补仓（如机器人、创新药、港股科技、沪深300）。
  - `核心观察（观望）`：**先看着，还没到买点**，在等条件转好后再评估（PEG降至合理/估值修复/催化增强等），不定死期限。
  - `核心观察（防御）`：**避险备用容器**，非日常低吸对象；只有市场全面转空、需要持币避险时才用（如黄金避险C、债、固收+）。
  - `核心观察（新候选）`：扩充的**低相关跨簇**标的（如红利低波、纳指），待「查潜力/资讯」诊断信号后再决定是否建仓。
  - `核心观察（估值恢复候选）`：（2026-07-23 优化）**当前不建仓，但条件恢复后可重新评估**；因极端估值/基本面暂时恶化等原因暂不可买，但若该标的回归合理区间(PE<50倍/估值分位<30%等)，可按正常「查潜力」流程重新评估是否纳入候选（如半导体013447——当前PE>100倍暂不碰，待估值回归后重新评估）。
  - 区别于 `候补`：候补是连每日盯都暂停了，需条件修复才重新纳入核心观察。
  - `已持·监控`：**已建仓的标的留在池内，仅盯卖点**（止盈 / 止损 / 减仓 / 提前卖出换弹药）与信号变化，**非建仓候选**；建仓逻辑锚点保留不删（与「移除规则」一致，不删除该行）。指令2「查潜力」不得把它当可建仓推荐。
  - **最新催化(资讯)列（动态·指令8留存）**：记录「指令8·资讯」扫描到的最新**正向 / 负向催化**（政策 / 业绩 / 资金流 / 行业利空），**带日期**，是每次资讯扫描的留存。与「观察原因」分工：**观察原因 = 当初为什么进池的静态逻辑**；**资讯列 = 现在又发生了什么的动态跟踪**。用法：正催化 → 交指令2 研判建仓/加仓；负催化 → 交指令3 研判提前卖/减仓；估值/动量变化 → 同步更新「当前信号」列。本列随「资讯」指令刷新，基线 2026-07-22；无新催化标「—」，若某日出现强负向催化须即时写入（如"XX政策转负 2026-MM-DD"）。
- **风险簇（关键 · 防伪分散）**：每只标所属「风险簇」。同属「**成长·科技·港股**」的（机器人 / 创新药 / 恒生科技 / 半导体 / AI / 创业板50）算**同一簇**，合并算单一敞口 **≤ 组合 45%**（已持 3 只已触顶，该簇暂停加仓）；**低相关**标的 = 沪深300（宽基锚）、黄金（防御）、红利低波（风格）、纳指（跨市场）、债 / 固收+（防御底仓），是真正的分散方向。**指令2「查潜力」与指令7「再平衡」须优先跨簇部署，绝不把闲置又撒回同簇。** 注：**同簇内所有标的（含半导体 / AI / 创业板50）的建仓均受 45% 上限约束——簇满则全部暂停**，技术面再好也不破此限；只有已持同簇止盈 / 减仓腾出额度后才可建。
- **移除规则 / 暂停观察（重要）**：**不要删除该行**，保留历史痕迹，靠**状态文字**区分，并对「基金（代码）」列加删除线弱化显示：
  - **已移除（永久退出观察）**：「基金（代码）」列加删除线 `~~…~~`，「备注」列写明 `已移除观察：<移除原因>`，状态改为 `已移除`。
  - **候补（暂移出每日观察，可恢复）**：「基金（代码）」列同样加删除线（表示当前不在活跃盯盘范围），状态保持 `候补（暂移出每日观察）`，备注写明恢复条件（如"动量修复再纳回"）；条件修复后取消该列删除线、状态改回 `核心观察`。
  - **删除线只标在「基金（代码）」一列**，其余列（信号/原因/状态/备注）保持可读，便于回看。
